Einführung in Markdown

Markdown GitHub Auszeichnungssprache

Was ist Markdown?

Markdown ist die nach HTML zweithäufigst genutzte Auszeichnungssprache der Welt. Sie wird unter anderem von GitHub, Reddit und Discord verwendet. Dies liegt vor allem an der einfachen Handhabung. Wenn du schon einmal auf GitHub unterwegs warst, hast du wahrscheinlich auch schon einmal diesen Dateinamen hier gesehen:
README.md
Die README.md-Datei in einem GitHub-Repo ist die lange Beschreibung. Sie erklärt, wofür das Projekt gut ist, wie man es verwendet, etc. Und sie muss in Markdown geschrieben werden!

Wie wird Markdown gerendert?

Markdown wird von einem sogenannten Preprocessor zu HTML umgeschrieben, welches dann im Browser auch so angezeigt werden kann.

Die Markdown-Syntax

## Betonung

Man kann Text mit *Asterisken* oder _Unterstrichen_ kursiv darstellen.
Verwenden man **doppelte Asterisken** oder __doppelte Unterstriche__, ist der Text fett.
Wenn du **doppelte Asterisken und _einfache Unterstriche_** verwendest, ist der Text zugleich fett und kursiv.
Für durchgestrichenen Text verwende ~~zwei Tilden~~ (auf der Tastatur AltGr + +)
Oder kombiniere **_~~alles~~_**.
            
## Listen
1. Erster Punkt einer geordneten Liste
2. Ein anderer Punkt
    * Ungeordnete Sub-Liste.
1. Die Zahlen sind eigentlich egal, es müssen nur Zahlen sein (Beim von GitHub favorisierten Markdown wird die Zahl angepasst)
    1. Geordnete Sub-Liste
4. Und noch ein Punkt.

    Man kann Absätze in Listenelementen richtig einrücken. Achte auf die leere Zeile oben und die Leerzeichen am Anfang (mindestens eins, aber wir verwenden hier drei, um auch den Markdown-Code auszurichten).

    Für einen Zeilenumbruch ohne Absatz musst du zwei Leerzeichen am Anfang verwenden.
    Beachte, dass diese Zeile separat ist, aber im gleichen Absatz.
    Das ist gegenteilig zum von GitHub favorisierten Zeilenumbruch, bei dem man keine Einrückung braucht.

* Für ungeordnete Listen kann man Sternchen verwenden
- Oder Minuszeichen
+ Oder Pluszeichen

## Code- und Syntaxhervorhebung
Codeblöcke gehören zu Markdown, aber Syntaxhervorhebung standardmäßig nicht. In Github-favorisiertem Markdown allerdings schon.
Inline-Codeblöcke kannst du so darstellen:

Inline `code` verwendet `Backticks`.

Codeblöcke werden durch drei Backticks umschlossen. Für Syntaxhervorhebung muss man die gewünschte Sprache in die obere Zeile nach den Backticks schreiben.

```javascript
// JavaScript-Syntaxhervorhebung
var content = document.getElementById("content");
function addContent(string){ // füge neuen Inhalt hinzu
    content.innerHTML += string;
}
```

```python
# Python-Syntaxhervorhebung
print("Milch", "Brot", "Wasser", "Honig", sep=", ") # erinnerst du dich?
```
