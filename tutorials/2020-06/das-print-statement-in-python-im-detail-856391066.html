3. Der Parameter flush

Kurze Version:
Der Parameter flush sorgt dafür, dass dein print-Statement sofort ausgeführt wird, unabhängig von end.
Lange Version: Wenn in Python das print-Statement ausgeführt wird, werden die Argumente nicht direkt gedruckt. Sie werden an ein IO-System (Input-Output) gedruckt. Und dieses hat in den meisten Fällen (glücklicherweise) einen Buffer. Dieser sorgt dafür, dass es nicht sofort gedruckt wird, sondern erst, wenn einer dieser Konditionen wahr wird:

  • \r oder \n im String
  • Ende der Python-Datei wurde erreicht
  • Buffer ist "voll" (das heißt, das maximale Zeichenlimit für den Buffer wurde erreicht)
  • Ein print-Statement mit flush=True wurde ausgeführt

4. Der Parameter file

Der Parameter file sorgt dafür, dass der Output nicht in die Konsole ausgegeben wird, sondern in eine andere Datei. Das Default ist hier sys.stdout (stdout für standard output, oder Standardausgabe). Zusammenfassung

  • Das print-Statement in Python hat vier optionale Argumente: sep, end, flush und file.
  • sep (engl. seperator) ist ein String, der die verschiedenen auszugebenden Argumente beim Ausgeben trennt. Normalwert " ".
  • end ist ein String, der am Ende angefügt wird. Normalwert "\n".
  • flush ist entweder True oder False, es sagt, ob der String unabhängig vom Buffer sofort zu gedruckt werden soll. Normalwert False.
  • file ist ein IO-Objekt, das den Output erhält. Normalwert sys.stdout, die Konsole.
