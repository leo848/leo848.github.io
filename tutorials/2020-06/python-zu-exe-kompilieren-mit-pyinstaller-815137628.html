Python zu exe kompilieren - mit PyInstaller

PyInstaller Pythonexe

Schritt 1

Zuallererst musst du dir PyInstaller installieren. Das ist recht einfach. Drücke Win + R, tippe in das sich geöffnete Fenster cmd ein und bestätige dies mit Enter.

Schritt 2

Das Fenster, welches du nun geöffnet hast, nennt sich Eingabeaufforderung. Du kannst damit zum Beispiel auch deine Python-Dateien ausführen. Um zu testen, ob du pip korrekt mit Python zusammen installiert hast, gib einfach pip in die Eingabeaufforderung ein. Wenn du Enter drückst, sollte dies hier etwa der Output sein:

Falls dies nicht der Output ist und du stattdessen etwas wie das hier siehst, solltest du erst einmal testen, stattdessen pip3 zu verwenden. Funktioniert dies auch nicht, solltest du Python neu installieren.

Schritt 3

Gib pip install pyinstaller ein. Am Ende sollte als Output entweder Successfully installed pyinstaller-x.x oder Requirement already satisfied: pyinstaller in ... (x.x) sein.

Schritt 4

Navigiere mithilfe von cd zu dem Ordner, indem sich eure Python-Datei befindet. Meine Python-Datei ist im Verzeichnis C:/Desktop/projects/quiz, also werde ich dorthin navigieren.

Schritt 5

Gebe nun das folgende ein:
pyinstaller --onefile [Relativer Pfad zur Datei, die du kompilieren möchtest]
Das --onefile sorgt in diesem Fall dafür, dass eine .exe-Datei erstellt wird. Ohne dieses Argument wird eine .exe-Datei mit vielen .dll-Dateien erstellt, was man meistens nicht möchte. Falls du eine Python-Datei hast, die nicht die Konsole nutzt (wie zum Beispiel eine GUI oder ein Nicht-Konsolen-Spiel), kannst du auch --noconsole als optionales Argument verwenden. Falls du ein Icon für deine .exe-Datei verwenden möchtest, nutze -i und dann den Pfad zur Datei. Sie sollte im .ico-Format sein, dazu gibt es Online-Converter wie ConvertICO.

Schritt 6

Es werden zuerst viele Zahlen und Wörter ausgegeben, aber am Ende sollte dies zu sehen sein: Es könnte natürlich eine andere Zahl zu sehen sein. Falls es allerdings einen Error gibt, liegt es in 90% aller Fälle an nicht kompatiblen Modulen. Im Verzeichnis der Python-Datei wirst du nun zwei neue Ordner und eine neue Datei sehen. Erst einmal eine Datei mit der Dateiendung .spec. Diese Datei brauchst du, wenn du erneut eine .exe erstellen möchtest, aber es schneller gehen soll. Falls du das nicht möchtest, kannst du sie löschen. Dann hast du zwei Ordner, build und dist. build kannst du getrost löschen, es sei denn, du hast einen Error. Dann wären dort alle Fehlermeldungen gelistet. Der Ordner, der uns interessiert, ist dist. In diesem befindet sich im --onefile-Modus eine .exe-Datei, falls du nicht --onefile verwendest, eine .exe-Datei und mehrere mit der Dateiendung dll. Wenn deine Python-Datei keine weiteren relativen Dateien verwendet, kannst du sie einfach so verwenden. Starte einfach die .exe und es sollte so sein, als startetest du die Python-Datei. Falls du aber Assets mit einem relativen Pfad hast, musst du die Anwendung in das Verzeichnis der Python-Datei verschieben. Dann sollte alles so funktionieren, wie es gedacht wurde.
